# Documentation Hub

This folder is organised by purpose so team members can quickly find the right
document for requirements, design work, implementation notes, reviews, and historical material.

> **Current status (2026-06-26):** The project has been reframed following a
> methodological audit (ML-AUDIT-2026-06-25). The original SAE-on-512-d strategy is
> now the baseline / failure case. Two new main methods are being implemented:
> **Path A** (SAE on 768-d hidden state) and **Path B** (SPLiCE). See the active
> strategy at `design/PROJECT-STRATEGY.md`.

## Main folders

- `requirements/` — project brief, evaluation guidelines, and source material.
- `design/` — active strategy (`PROJECT-STRATEGY.md` v2.0), implementation plan (`IMPLEMENTATION-PLAN.md` v3.0), and proposal documents.
- `plans/` — detailed implementation plans for specific tasks and features.
- `audits/` — review and audit reports.
- `releases/` — changelogs and release history (updated after a plan is implemented).
- `wiki/` — reference knowledge base and module documentation.
- `archive/` — retired documentation kept only for traceability.
