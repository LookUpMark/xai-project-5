# Documentation Hub

This folder is organized by purpose so team members can quickly find the right place for requirements, design work, implementation notes, reviews, and historical material.

## Main folders

- `requirements/` - project brief, evaluation guidelines, and source material.
- `design/` - strategy, implementation plan, implementation notes, and proposal documents.
- `implementation/` - runbooks, execution notes, and operational checklists.
- `audits/` - review and audit reports.
- `releases/` - changelogs and release history.
- `wiki/` - reference knowledge base and module documentation.
- `archive/` - retired documentation that should be kept only for reference.
