## What Changed

### 1. Contract reconciliation (producer aligned to judge)

The SAE producer previously emitted `findings` / `sample_idx` / `naming_confidence`, which caused
3 `KeyError`s in the judge's pair-building loop. The judge and `docs/draft/plan.md:371-393` agree
on a different schema, so the producer was aligned to that canonical form.

- `contracts.py`: `Finding` → `ConceptActivation` (`feature_id, name, activation`);
  `Explanation` → `{image_id, top_k_concepts, pseudo_report}`. Schema-doc only (not runtime-enforced).
- `generate_explanations.py`: `generate_explanation()` returns `top_k_concepts` + `pseudo_report`;
  `run()` loads `test_image_ids.json` and stamps `image_id` per record (fallback `sample_{idx}`).
- `autoencoder/__init__.py`: re-exports `ConceptActivation` instead of `Finding`.

### 2. `image_id` propagation chain

The bare `(N, 512)` embedding tensors carry no row metadata, so identity is propagated via a
sidecar JSON kept in lockstep with the tensors:

| Stage | File | Behavior |
|-------|------|----------|
| Extraction | `src/extract_embeddings.py` | Captures `batch_paths` (previously discarded), writes `visual_image_ids.json` (basename per row) next to `visual_embeddings.pt` |
| Config | `src/config.py` | Adds `visual/train/test_image_ids_path` to `PathsConfig` (under `embeddings/`) |
| Split | `src/utils.py` | New `_split_ids()`; `split_embeddings()` gained `source_ids_path` / `train_ids_path` / `test_ids_path`, sliced with the **same** `train_idx`/`test_idx` permutation |
| Split (CLI) | `src/autoencoder/train_sae.py` | `prepare_split()` calls `_split_ids` when the visual sidecar exists (parallel to `split_embeddings`, same seed) |

**Design guards:** id files regenerate only on a *fresh* split (the tensor skip-if-exists guard
also short-circuits id splitting — delete the split `.pt` files to refresh ids); `_split_ids`
raises `ValueError` if the sidecar length ≠ train+test rows (catches misalignment).

### 3. VLM extraction notebook fixed to the local layout

`notebooks/vlm/extract_embeddings.ipynb` was configured for another machine's Kaggle-extracted
layout (`data/iu_xray/chest-xrays-indiana-university/`, `indiana_reports.csv` in root, `cuda`).
Repainted to the local layout that matches the default `EmbeddingConfig`:

- `DATASET_ROOT = data/iu_xray` (was `data/iu_xray/chest-xrays-indiana-university`)
- `reports_dir = data/iu_xray/reports` (was the extraction root)
- staging `reports_csv = data/iu_xray/reports/indiana_reports.csv`

No source files were modified for this — only the notebook, per scope.
