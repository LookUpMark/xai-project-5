## Findings

### [F-001] Stability still at the analytical chance floor — non-identifiability persists
- **Severity:** HIGH (scientific)
- **Evidence:** mean cross-seed Jaccard 0.0083, std 0.0006, analytical floor k/(2D−k) = 0.0079
  (D=2048, k=32). Observed/floor ratio = **1.05×** — i.e. essentially at chance.
- **Honest framing:** the "2.2× lift over baseline 0.0038" quoted in `REPORT_stability.md` is
  **not** evidence of identifiability — it is mostly the smaller dict_size raising the floor
  (baseline floor at D=4096 was 0.0039; observed 0.0038 = 0.97× its own floor). Both Path A and
  the baseline sit at their respective chance floors. **Moving to 768-d did not, by itself, make
  the sparse factorisation identifiable across seeds.**
- **Root cause (per audit M-002):** 5955 train images for a 2048-d dictionary ≈ 2.9 samples/feature;
  SAE/CLIP literature operates at 10⁵–10⁶ samples. Data scale remains binding.
- **Remediation options:** (a) shrink dict (1024 → ~5.8 samples/feat); (b) accept Path A as a
  feature extractor and evaluate it on faithfulness (LLM judge) rather than seed-stability;
  (c) more data. Do **not** add more seeds — they will not help.

### [F-002] Dead-feature rate and naming both improved (the wins)
- **Severity:** — (positive)
- **Dead features:** activation-based 12.4–14.1% (mean 12.9%) vs baseline 40–60%; decoder-norm
  dead 0.0%. The 768-d pre-projection representation is far less degenerate than the projected one.
- **Naming:** mean 0.4711 (median 0.4709), +0.076 over baseline SAE (0.395), +0.099 over random
  (0.372). The frozen-projection bridge is sound. (Top terms are broad RadLex anatomy — e.g.
  *C5 vertebral body*, *large intestine* — clinically loose for chest X-ray, so naming is above
  random but not sharp; the RadLex dictionary breadth limits ceiling.)

### [F-003] Pre-existing vocab/embedding mismatch (fixed, but flag for baseline)
- **Severity:** MEDIUM (data integrity, pre-existing)
- **Evidence:** `data/vocabulary.json` had **1030** terms but `embeddings/standard/text_vocab_embeddings.pt`
  had only **508** rows. This would silently break `concept_naming.py` too (it asserts equality).
- **Action taken:** regenerated vocab embeddings from the current 1030-term vocabulary
  (`get_text_features`, L2-normalized); old 508-row file backed up as `text_vocab_embeddings.bak508.pt`.
  Labels and embeddings now align (1030 = 1030). The baseline `concept_names.json` in `results/`
  predates this and is stale relative to the current vocabulary.

### [F-004] Reconstruction cosine is near-saturated and uninformative (M-004, reaffirmed)
- **Severity:** LOW (interpretation)
- Test cosine 0.97–0.97 across all seeds. Per M-004 this carries ~no signal about feature
  quality; it is reported only for completeness, not as evidence the SAE works.

## Conclusion

### Overall Status: ✅ PASSED (implementation) / ⚕️ PARTIAL (scientific)

The deliverable requested — a correctly-organised, runnable Path A pipeline whose scripts each
emit a markdown REPORT, free of logic errors — is **complete and verified**. The frozen-projection
naming bridge is novel, tested, and correct.

The **scientific** outcome is honest and mixed: Path A clearly improves dead-feature rate and
concept naming over the 512-d baseline, but **does not escape the cross-seed non-identifiability
that motivated it** — Jaccard remains at the analytical chance floor. This is itself a valid,
reportable result: it localises the failure to **data scale (M-002)**, not representation choice,
and it does not contradict the project strategy (Path A was the centrepiece *attempt*; the audit
explicitly flagged scale as a confound).

### Recommendations
- Do **not** treat cross-seed Jaccard as the pass/fail metric for Path A at this scale; evaluate
  via the LLM-judge faithfulness axis (next stage) and against Path B (SPLiCE).
- If stability matters, run a dict_size sweep (1024 / 1536) and report observed/floor ratio, not
  raw Jaccard.
- Commit the regenerated 1030-term vocab embeddings; delete `text_vocab_embeddings.bak508.pt`
  once the baseline is re-run against them.
