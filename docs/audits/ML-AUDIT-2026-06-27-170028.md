### 🔴 F-003 — `clamp(min=0)` destroys the OMP reconstruction (8–14× error inflation)
| Field | Value |
|---|---|
| Severity | **S0 / RED** |
| Category | model / evaluation correctness |
| Location | `src/concept_discovery/spliece.py:53-54` |
| Evidence | Measured on real images (k=32): reconstruction error `‖emb−Vᵀc‖` **0.146 → 2.03 (13.9×)** with gap; **0.53 → 4.35 (8.1×)** without gap. ~14–15 of 32 selected atoms are legitimately negative and get zeroed → result no longer OMP-optimal, residual exceeds the zero-vector. Clamped-topk agrees with true `\|coef\|` OMP support on only ~58% of atoms → the ranking shipped to the judge is ~42% a different concept set. |
| Impact | The "sparse decomposition = explanation" identity is broken; `% Aligned` grades a corrupted decomposition, inflating/deflating faithfulness in an uncontrolled way and invalidating the cross-method comparison. |
| Remediation | Enforce non-negativity **inside** the solver: `sklearn.linear_model.Lasso(positive=True)` on `X.T` (already a dependency; **matches `PROJECT-STRATEGY.md` which specifies Lasso(positive=True)** — the impl diverged from its own design doc), or NN-OMP. Drop the clamp. Effort **M**. |
| Status | OPEN |

## 7. Positive Confirmations (GREEN / S4)

- **G-001:** OMP transpose convention is **correct** — `omp.fit(X.T, y)` with `X.T=(512,V)` solves `emb ≈ Σ cᵢ·vocab_emb[i]`; hand matmul bit-identical to sklearn's internal reconstruction (`max|predict−X.T@coef|=0`). Coefficients map to the right atoms.
- **G-002:** Fully **deterministic** — byte-identical across runs and BLAS thread counts; committed output reproduces from current code+inputs.
- **G-003:** `scikit-learn==1.8.0` exact-pinned; only public OMP API used.
- **G-004:** dtype/device clean — CPU-only correct (`load_tensor` forces `map_location="cpu"`); float32 consistent; the float64→float32 cast is negligible next to the clamp error.
- **G-005:** `modality_gap.pt` provenance clean (train-only, no test leakage).
- **G-006:** Vocabulary is dataset-independent (RadLex + fixed seeds).
- **G-007:** Train/test split is study-group-aware with an explicit leakage guard.
- **G-008:** `SpliCEConfig` `@dataclass(frozen=True)` + `field(default_factory=...)` pattern is correct.
- **Bonus:** gap correction does real beneficial work (max cosine 0.44→0.83; pre-clamp residual 0.53→0.15). The decomposition problem is entirely the clamp, not the gap.
