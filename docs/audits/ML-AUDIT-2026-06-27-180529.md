## ML-section health

**Data pipeline health — GOOD.** No train/test leakage in either method: `utils.split_embeddings` groups by radiograph study, yielding **0** overlapping studies between train/test (3081 train / 771 test) in both 512-d and 768-d. Critically, both methods partition the **same** studies (`set(512 test ids) == set(768 test ids)`, same train), so baseline-vs-Path-A compares on identical held-out images. Naming never touches test; the modality gap is fit on train only. Explanations are generated from TEST embeddings (`explanation_max_samples=None`). Tensor shapes throughout (bridge `W_dec(2048,768)@W_proj.T(768,512)→(2048,512)`, gap broadcast, topk dim=1, decoder transpose) are all correct; dtype is float32 end-to-end. **Baseline `sample_explanations.json` is judge-ready** (1515, clean schema, all names populated, 0 `unknown_feature_`).

**Model architecture assessment — SOUND.** dict_size=2048 for **both** methods (the stale 1024 in CLAUDE.md prose is gone; only CLAUDE.md text is out of date). LR schedule, seed handling, init, and gradient hygiene are textbook TopK (Gao et al. 2024). The frozen-projection naming bridge (`visual_projection` confirmed frozen+detached, `(512,768)`) is the architecturally correct way to name 768-d pre-projection features against a 512-d text vocab — mean naming cosine 0.471 > random 0.372 and > baseline 0.395. The two real model defects (F-007 dead-mask structural zero, F-008 AuxK disabled) do not corrupt the current per-sample explanations. The matched (permutation-invariant) stability metric works and is not degenerate (slot-wise Jaccard is the degenerate one, correctly flagged elsewhere).

**Evaluation reliability — MIXED.** Schema contract with the judge is satisfied by both files (interface audit: 0 violations). But comparison validity is threatened by F-004 (null k mismatch), F-005 (no fresh-run dedup), F-006 (denominator), F-012 (no CI), and above all F-001 (the 2× Path-A denominator). The `verdict` column is the correct numerator (HANDOFF was right that `aligned_score` does not exist); F-006 is the only nuance.
