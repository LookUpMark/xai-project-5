# Docs Overview

This folder is the entry point for the documentation set used to hand off work between team members.

## How to navigate

- `requirements/` - project brief, evaluation guidelines, and source material.
- `design/` - strategy, implementation plan, implementation notes, and proposal documents.
- `implementation/` - operational notes, runbooks, and execution-focused documentation.
- `audits/` - review and audit reports.
- `releases/` - changelogs and release history.
- `wiki/` - reference knowledge base and module documentation.
- `archive/` - retired or superseded documentation.

## Suggested reading order

1. Start with `requirements/PROJECT-BRIEF.md`.
2. Read `requirements/EVALUATION-GUIDELINES.md`.
3. Review `design/PROJECT-STRATEGY.md` and `design/IMPLEMENTATION-PLAN.md`.
4. Check `design/proposals/` for exploratory notes and alternative approaches.
5. Use `audits/` and `releases/` for historical context.

## Maintenance rule

When adding a new document, place it in the narrowest folder that matches its purpose. If a document becomes obsolete, move it to `archive/` instead of leaving it in place.
