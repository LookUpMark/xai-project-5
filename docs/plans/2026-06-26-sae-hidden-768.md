## Overview

Implement the **centrepiece method** of the project: train a Top-K SAE on BiomedCLIP's
**768-d CLS token of the last hidden state, *before* the vision projection head**, instead of
the 512-d projected embedding that the (failed) baseline used. Build a separate naming bridge
to map SAE decoder rows back into the 512-d RadLex text space. Verify against the baseline's
analytical-null numbers (Jaccard 0.0038, dead 40–60%, naming ≈ 0.372).

This is **additive**: the 512-d baseline config and scripts are left untouched. Path A lives in
its own dirs and a new `src/sae_hidden/` package that reuses the existing `SAEManager`, `utils`,
and `IUXrayImageDataset`.

## Approach Justification (literature + audit)

- **M-001 (audit):** the baseline fails because a learned, seed-dependent SAE on the 512-d
  projected space is a *non-identifiable* sparse factorisation in an anisotropic,
  information-bottlenecked space. Fix option (b): keep the SAE, move it to the **768-d
  pre-projection hidden state**.
- **SAE-on-CLIP literature** (audit §"Paradigm B", arXiv 2504.08729) trains SAEs on
  **residual-stream / hidden activations**, not the final projection. Path A aligns with that.
- **Naming-bridge trade-off (audit M-001):** decoder rows move to 768-d, RadLex text lives in
  512-d. We resolve it by projecting decoder rows through the **frozen `visual_projection`
  (512×768, no bias)** → 512-d, then cosine vs RadLex. This is the audit's option (a): reuse the
  frozen projection, not a learned probe.

**Source-of-truth checks performed against the installed model** (`chuhac/BiomedCLIP-vit-bert-hf`,
`modeling_biomed_clip.py`):
- `model.vision_model` (line 652) returns `last_hidden_state` (B,197,768); CLS = `[:, 0, :]` (line 609).
- `model.visual_projection = nn.Linear(768→512, bias=False)` (line 654); `.weight` is (512,768).
- `get_image_features` = `visual_projection(pooled_output)` (line 755) → baseline is post-projection, confirmed.

## Hyperparameters (audit-corrected)

| Param | Baseline (512-d) | Path A (768-d) | Why |
|---|---|---|---|
| `activation_dim` | 512 | **768** | pre-projection hidden state |
| `dict_size` | 1024 (cfg) / 4096 (doc) | **2048** (recommended) | M-002: 4096 ⇒ 1.5 samples/feat; 2048 ⇒ 2.9, 2.7× overcomplete |
| `k` (Top-K) | 32 | 32 | unchanged |
| `lr` | 5e-5 | 5e-5 | M-006: explicit low LR for small data |
| `steps` | 50,000 | **8,000** | M-006: 50k = ~2,140 epochs (overfit) |
| `batch_size` | 256 | 256 | unchanged |
| `seeds` | (0,42,123,456,789) | same | stability analysis |
| normalize input before SAE | yes (L2) | **no (raw)** | SAE-on-residual-stream lit trains on raw activations |

`dict_size` and the normalize flag are the two open decisions (see HARD GATE).

## Implementation Steps

### Step 1 — Config (`src/config.py`)
- Add frozen `SAEHiddenConfig(activation_dim=768, dict_size=2048, k=32, lr=5e-5, steps=8000, batch_size=256)`, validated in `__post_init__`.
- Add hidden paths to mutable `PathsConfig.__post_init__`: `hidden_embeddings_dir` (`embeddings/standard_hidden`), `hidden_visual/train/test_embeddings_path`, `hidden_models_dir` (`models/sae_hidden_seed{N}` base), `hidden_results_dir` (`results/sae_hidden`). Reuse `text_vocab_embeddings.pt` (512-d) and `modality_gap.pt` from baseline dir.
- **Verify:** `config.SAEHiddenConfig().activation_dim == 768`.

### Step 2 — Extraction (`src/sae_hidden/extract_hidden.py`)
- `extract_visual_embeddings_hidden(model, processor, dataset, config) -> (Tensor(N,768), list[str])`: batch images → `model.vision_model(pixel_values=…)` → `last_hidden_state[:,0,:]` → **raw** (no per-sample L2 norm for the SAE input). Save `visual_embeddings_768.pt` + `visual_image_ids.json`.
- Split via existing `utils.split_embeddings` (group-aware, no leakage) → `train_embeddings_768.pt` / `test_embeddings_768.pt` (+ sidecars).
- Writes `results/sae_hidden/REPORT_extraction.md`: counts, shapes, per-split norm stats, confirmation CLS dim = 768.
- **Verify:** output `(N,768)`; train/test share no study (reuse anti-leak assertion in `split_embeddings`).

### Step 3 — Training (`src/sae_hidden/train_hidden.py`)
- Loop seeds; build `SAEManager({"activation_dim":768,"dict_size":2048,"k":32,"lr":5e-5,"steps":8000,"batch_size":256})`; `mgr.train(train_768, seed)` → `models/sae_hidden_seed{N}/trainer_0/ae.pt`.
- After each seed: sanity metrics on **test_768** (MSE, cosine, sparsity, dead%).
- Writes `results/sae_hidden/REPORT_training.md`: per-seed loss, reconstruction, dead-feature %, L0.
- **Verify:** 5 model dirs exist; `activation_dim` from `encoder.weight` == 768.

### Step 4 — Naming bridge (`src/sae_hidden/naming_hidden.py`)
- Load primary seed (42) decoder `W_dec` (dict,768) via `mgr.get_decoder_weights()`.
- `W_proj = model.visual_projection.weight.detach()` (512,768). `dec_512 = W_dec @ W_proj.T` → (dict,512); L2-normalise; subtract `modality_gap` (512-d); cosine vs RadLex `text_vocab_embeddings.pt` (V,512).
- Dead-feature flagging (zero-norm decoder) reused from baseline pattern.
- Writes `results/sae_hidden/concept_names.json` + `REPORT_naming.md`: naming mean vs random (0.372) and baseline SAE (0.395); top concepts.
- **Verify:** `dec_512.shape == (dict,512)`; naming mean > 0.372.

### Step 5 — Stability (`src/sae_hidden/stability_hidden.py`)
- `SAEManager.compute_stability(model_dirs=[5 hidden seeds], embeddings=test_768)` → pairwise cross-seed Jaccard.
- Writes `results/sae_hidden/stability_analysis.json` + `REPORT_stability.md`: mean Jaccard vs baseline (0.0038) and analytical null `k/(2D−k)`; dead-feature rate vs 40–60%.
- **Verify:** Jaccard meaningfully above 0.0038.

### Step 6 — Explanations (`src/sae_hidden/generate_explanations_hidden.py`)
- `mgr.get_top_concepts(test_768, n=k)` → per-sample pseudo-report using `concept_names.json`.
- Writes `results/sae_hidden/sample_explanations.json` + `REPORT_explanations.md` (sample preview + stats).
- **Verify:** JSON valid; top concepts map to named (non-dead) features.

## Risks and Mitigations

| Risk | Prob | Impact | Mitigation |
|---|---|---|---|
| Extraction slow on CPU (7,470 imgs) | High | Med | Batch; flag GPU; script is one-off, output cached |
| dict_size too large → still non-identifiable (M-002) | Med | High | Default 2048 (not 4096); expose in config |
| Normalising 768-d input hides magnitude (lit says raw) | Med | Med | Default raw; flag as ablation |
| modality_gap is the 512-d baseline gap, not recomputed | Low | Low | Reuse valid; gap is in 512-d space the bridge lives in |
| `chuhac` mirror attribute drift | — | — | Already verified against installed source; assert at load |
