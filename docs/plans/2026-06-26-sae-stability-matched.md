# Implementation Plan: Permutation-Invariant Cross-Seed Stability (Path A fix for F-001)

> **Version:** 1.0
> **Status:** Draft (awaiting approval)
> **Generated by:** ml-implementation-guardian
> **Branch:** `feat/sae-hidden-768`
> **Fixes:** ML-AUDIT-2026-06-26 finding **F-001** (cross-seed Jaccard at chance floor, mis-interpreted as non-identifiability)

## Overview

The current `SAEManager.compute_stability` measures **slot-wise index Jaccard** (feature #342 vs
#342 across seeds). Because SAEs have no canonical feature ordering, this metric is ~0 by
construction even for *identical* dictionaries up to permutation. F-001's claim "the SAE is
non-identifiable" therefore **over-interprets** a metric that cannot show identifiability in the
first place — an interpretation error of the same class as ML-AUDIT-2026-06-23 (contamination)
and 06-25.

This plan adds a **literature-grounded, permutation-invariant** cross-seed feature-agreement
metric alongside the existing one, and reframes the result in the literature's own terms
(weak vs strong universality). It does **not** remove the slot-wise metric — it labels it
correctly and demotes it.

## Approach Justification (literature)

Canonical cross-seed / cross-model SAE feature comparison (the exact problem we have):

| Step | Literature source | What it does |
|---|---|---|
| 1. Permutation-invariant feature **matching** | Bricken et al. 2023 (*Towards Monosemanticity*); Lan et al. 2024 (2410.06981, §3) | For each feature in seed A, pair it with its **most similar** feature in seed B. Solves the permutation problem our slot-wise metric ignores. |
| 2. Global rotation-invariant metric (SVCCA/RSA) on matched subsets | Raghu et al. 2017; Lan et al. 2024 | Measures subspace alignment beyond pairwise matches. **(Optional extension — see Scope.)** |
| 3. Null via **random pairing permutation test** | Kriegeskorte et al. 2008; Lan et al. 2024 §3 | N random pairings → null distribution → p-value. The proper analog of our closed-form `k/(2D−k)` floor, done as the literature specifies. |
| Framing | Leask et al. 2025 (*"SAEs do not find canonical units of analysis"*); Lan et al. 2024 App. E.2, H | Cross-seed SAEs exhibit at most **weak universality** (shared subspaces), rarely **strong universality** (identical features). Not finding identical features is *expected*, not a bug. |

**Matching metric choice: decoder-direction cosine (not activation correlation).** Bricken's
primary method is activation correlation, but on our data it is pathological: TopK k=32 over
1,515 test samples means each feature is active on ~2% of samples, so Pearson correlation is
dominated by shared zeros. **Decoder cosine** (L2-normalised `W_dec` rows) is the SAE *feature
direction* — what a feature *means* — and is explicitly validated by Lan et al. 2024 (App. E.2)
for the **same-model-different-seed** case (our exact case): *"we also use cosine similarity
instead of activation correlation… average cosine 0.9… SVCCA 0.92."* Decoder cosine avoids the
sparsity pathology and is literature-endorsed for this setting.

## Metric specification (the core)

For each unordered seed pair (i, j), loading one model at a time (reuse the memory-safe `del +
empty_cache` pattern from `compute_stability`):

1. **Decoder matrices**: `W_i, W_j` ∈ ℝ^{D×768} via `SAEManager.get_decoder_weights()`.
2. **Drop dead features** from both (reuse `sae_hidden.naming_hidden.dead_feature_mask`;
   audit F-002 says decoder-norm dead = 0.0%, so currently a no-op but guards future runs).
3. **L2-normalise rows**; compute cosine matrix `C = Ŵ_i Ŵ_jᵀ` ∈ ℝ^{D×D}.
4. **Best-match** for each feature a in i: `m_a = max_b C[a,b]`. Record the matched index.
5. **Report** (mean over all D live features, then mean over pairs):
   - `mean_best_match_cosine`
   - `frac_matched_>=0.7`, `frac_matched_>=0.9`  (cf. Lan et al. "X% of feature pairs kept")
   - `frac_mutual_1to1`  (a↔b is each other's argmax — cleaner, less noisy; Lan et al. App. J)
6. **Null (permutation test)**: shuffle B's rows, recompute `mean_best_match_cosine`, repeat
   `N=200` → null mean + **p-value** = fraction of null runs ≥ observed.
   - *Analytical sanity anchor* (secondary): E[max cosine] ≈ (1/√768)·√(2·ln D) ≈ 0.14 for D
     random unit vectors. The permutation null is primary because it controls for the data
     manifold concentration (cf. naming random baseline 0.372, F-002).

**Acceptance / interpretation rule (written into the report so it cannot be re-misread):**
- observed `mean_best_match_cosine` **≫ null** with p<0.05 and `frac_matched_>=0.7` well above
  the null → **weak universality present** (Path A features reproduce up to permutation).
- observed ≈ null, p>0.05 → **genuine non-identifiability at this data scale** (M-002
  confirmed, now *measured* rather than asserted).

## Verification Plan

Unit tests (synthetic 768-d tensors, no real models — fast, like the existing 10):

- `test_matched_identity_is_one`: a decoder vs itself → `mean_best_match_cosine == 1.0`,
  `frac_matched_>=0.9 == 1.0`, p-value ≈ 0.
- `test_matched_permutation_invariant`: permute B's rows → identical best-match distribution
  (the whole point: this is what slot-wise Jaccard fails).
- `test_matched_random_is_at_null`: two independent random decoders → observed ≈ permutation
  null, p>0.05 (non-significant). Confirms the null works.
- `test_matched_dead_excluded`: a zero decoder row is dropped, not matched.

End-to-end (real IU X-Ray, MPS, <1 min): `stability_hidden.run_matched()` over the 5 trained
seeds → `stability_matched.json` + `REPORT_stability_matched.md`. Existing 5 models reused —
**no retraining**.

## Risks and Mitigations

| Risk | Prob | Impact | Mitigation |
|---|---|---|---|
| Decoder-cosine inflated by manifold concentration (random RadLex already ≈0.37) | Med | Med | **Permutation null is primary** (controls for concentration); report observed/null ratio, not raw cosine. |
| D×D matrix (2048²≈16 MB/pair) on MPS | Low | Low | Run on CPU, float32; trivial at this size. |
| Activation-correlation chosen by mistake (sparse pathology) | — | — | Decision already made: decoder cosine. Documented in plan + LITERATURE doc. |
| Result still at null → looks like "another failure" | Med | Low | Reframe honestly: that is the *correct* measurement of M-002 (data scale), now distinguishable from a metric artifact. Either outcome is publishable. |
| SVCCA omitted → "not fully literature-aligned" | Med | Low | Core uses the literature-*primary* metric (matching + null, Bricken/Lan). SVCCA/RSA documented as the optional global layer (see Scope). |

## Scope decision (needs your call)

- **Core (recommended):** decoder-cosine matching + permutation null + p-value + mutual-1to1.
  Directly fixes F-001, literature-primary, ~1 method + 1 wiring + tests. Smallest correct fix.
- **+ SVCCA/RSA global layer (full Lan et al. parity):** adds a rotation-invariant subspace
  metric on matched subsets. Bigger lift (SVCCA impl or `mvlearn` dep), arguably a separate
  concern. Recommended as a follow-up, not part of "the last fix."
