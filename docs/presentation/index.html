Part 1 of 3 · Nicolò Colle

The problem

Medical VLMs are accurate but opaque.

Strong performance on pathology classification, segmentation and report generation — but the internal representations are high-dimensional, polysemic and unreadable. A serious concern in safety-critical clinical use, where classic post-hoc tools (saliency, attention) rarely resolve into semantic structure.

Ref: main.tex §1 Introduction

Accuracy ≠ understanding. In radiology, a model right-for-the-wrong-reasons is as dangerous as one that is simply wrong.

Saliency/attention show where the model looks, not what it knows in human-readable form. That gap motivates concepts.

Part 1 of 3 · Nicolò Colle

Approach

Concept-based XAI: from supervised to unsupervised discovery.

Supervised

Concept Bottleneck Models (Koh 2020) and TCAV (Kim 2018) need a pre-defined, labelled concept set — exactly the limitation unsupervised discovery targets.

Sparse Autoencoders

Bricken et al. (2023): dictionary-learning on activations yields monosemantic latents. The Top-K SAE (Gao 2024) sets L₀ exactly via top-k gating. We adopt it.

MedConcept (Haque 2026)

SAE extraction → alignment to a radiology vocabulary → an LLM-judge with Aligned/Unaligned/Uncertain metrics. The template we re-implement and stress-test.

Impl: src/autoencoder/, src/vocabulary_building/, src/evaluate_llm_judge.py

Two families: supervised concepts (limited by labelled sets) vs unsupervised discovery (decompose + name).

We inherit the MedConcept paradigm (3 stages) but evaluate rigorously: null-calibrated metrics, permutation-invariant statistics.

SPLiCE (Bhalla 2024) — a deterministic third way — detailed in Part 3.

Part 1 of 3 · Nicolò Colle

Research gaps · in the paper's order

Six gaps this work addresses.

GAP 1

Instability / non-identifiability

Sparse factorisations are not reproducible across seeds; no prior medical-VLM work reports seed-stability against a null.

GAP 2

Clinical validity

A cosine-assigned name is cosmetic unless it tracks real pathology.

GAP 3

Representation-location mismatch

SAE literature uses raw hidden states; reference pipelines fit on the already-projected, gap-bearing CLIP space.

GAP 4

Small-data robustness

SAEs need 10⁵–10⁶ activations; medical corpora are 2–3 orders smaller → dead features.

GAP 5

Flat concepts

Explanations are top-k lists that ignore anatomical / hierarchical structure.

GAP 6

Non-reproducible evaluation

Claims rest on single metrics whose chance floor is never reported; slot-wise overlap is degenerate under per-seed permutation.

Ref: main.tex §3 Research gaps

Roadmap: every later experiment answers ≥1 of these gaps, tagged on each slide's eyebrow.

Gap order is exactly the paper's §3. Results follow the paper's experiment sequence (§5), not strict 1→6 — the paper itself proves the metric (Gap 6) before using it.

Part 1 of 3 · Nicolò Colle

Backbone & vocabulary

BiomedCLIP + RadLex.

Our backbone is BiomedCLIP. Its image encoder exposes a 768-d hidden state that a frozen projection compresses into the 512-d contrastive space we actually decompose. We name features against a curated RadLex radiology vocabulary. And we correct the modality gap — the systematic offset between the image and text cones — before any naming or decomposition.

Impl: src/embedding_extraction/extract_embeddings.py, src/vocabulary_building/build_vocabulary.py

Key point: 768-d (hidden) → 512-d (contrastive). Baseline operates on 512-d; Path A goes back to 768-d pre-projection.

The modality gap is not a detail — correcting it lifts max atom–image cosine 0.49→0.85 (Part 3). It was the binding issue, not the solver.

Data

Two scales: IU X-Ray and ROCOv2 (10×).

IU X-Ray · 1× · primary

The small-scale setting.

Split by radiographic study, so no study appears in both train and test — verified zero overlap, recomputed deterministically every run.

ROCOv2 · 10× · replication

A ten-times larger corpus.

Split by image. It exists to test one hypothesis head-on: does more data cure non-identifiability?

Impl: src/utils.py (split_embeddings) · anti-leakage test: tests/unit/test_split.py

IU split by study (no patient leakage), ROCOv2 by image (no shared structure). Deterministic, recomputed each run.

The 10× is the test bed for the scale hypothesis — spoiler: it does not help.

Method · three paths

Three decompositions, one SAE config.

01 · BASELINE

Top-K SAE 512-d

Faithful MedConcept re-implementation on the projected space. Trained across multiple seeds — that is what makes a stability metric possible. (Gap 1)

02 · PATH A

Top-K SAE 768-d

CLS hidden state before projection; bridge with frozen W_proj. Tests the representation site. (Gap 3)

03 · PATH B

SPLiCE

Decomposition over a fixed RadLex dictionary — no learning, so deterministic. (Gap 1+4)

Impl: src/autoencoder/sae_module.py (SAEManager), src/sae_hidden/, src/concept_discovery/spliece.py

One SAE config for clean comparisons: D=2048, k=32, 8k steps, 5 seeds (0,42,123,456,789). Multi-seed enables stability.

Each path targets explicit gaps (tagged). Path B changes paradigm: fixed dictionary, not learned.

Part 2 of 3 · Marc'Antonio Lopez

Evaluation · Gap 6

A matched metric, against a subspace null.

Stability must be permutation-invariant: for each decoder row of one seed, take the best cosine match across the other seed. The null is conditioned on the decoder's effective rank (~357, not the ambient 512) — that controls for how concentrated the data manifold is. The old slot-wise Jaccard is not permutation-invariant, so it is degenerate (proved next).

Impl: src/autoencoder/stability_analysis.py, src/sae_hidden/naming_hidden.py (compute_random_baseline)

The metric is the contribution. Slot-wise seems reasonable but is degenerate under feature permutation — we prove it on the next slide.

The subspace-conditioned null is the honest part: real decoder directions live in ~357-d, not 512-d. An isotropic null would inflate the ratio.

The headline result · Gap 1
1.81×1.67× on IU X-Ray · 1.81× on ROCOv2

The baseline SAE reconstructs its inputs well — but cross-seed feature identity is absent: best-match barely above the conditioned null, almost nothing matches strongly, naming stays low. Genuine non-identifiability, not a training bug.

Pivot slide. Good reconstruction ≠ canonical features.

Not a bug: decoders full-rank, inits distinct, no collapse. Intrinsic to the representation site (pooled CLS, compressed, contrastive-shaped).

Numbers (Tab.1): matched obs 0.299 (IU) / 0.327 (ROCOv2); subspace null ratio 1.67× / 1.81×; frac ≥0.9 0.0% / 0.3%; naming 0.40 / 0.48.

Part 2 of 3 · Marc'Antonio Lopez

Relabeling control · Gap 6

The slot-wise metric is floor-by-construction.

Rename one SAE's features with a random permutation — same network, same reconstruction — and the slot-wise Jaccard collapses to the 0.0077 floor, indistinguishable from two genuinely different SAEs. The matched metric correctly sees identity. Slot-wise is permutation noise, not a stability measure.

Impl: scripts/relabeling_control.py — relabel_state_dict(sd, perm)

Cross-seed Jaccard matrix (IU X-Ray baseline). Mean ≈0.0077 — the analytical floor.

Trap experiment: a relabelled SAE is mathematically identical to itself, yet slot-wise Jaccard does not recognise it. So slot-wise is permutation noise.

Contribution (1) of the paper. From here, every stability claim uses the matched metric.

Moving the representation site is not enough.

Path A trains the SAE on the hidden state before the projection. It is genuinely healthier — far fewer dead features, better naming — but the permutation-invariant verdict does not change: still almost no features match strongly. This is weak universality: a shared subspace, not identical features. It still operates on the pooled CLS token, not patch tokens.

The scale hypothesis · Gap 4
10×ROCOv2 ~80k vs IU X-Ray ~7k

Ten times more data improves training health but not identifiability — every stability indicator stays flat. The binding constraint is the representation site, not data volume.

Contribution (3): direct refutation of "more data = identifiable features".

10× moves training-health, not identity. erank ≈357–363 ≪ 2048-way dictionary → no canonical feature on a compressed global vector.

Part 3 of 3 · Carmine Francesco Benvenuto

Path B · SPLiCE · Gap 1+4

Deterministic. Judge-ready.

SPLiCE decomposes each embedding into a sparse, non-negative combination over the fixed RadLex dictionary. The dictionary is never estimated, so the result is deterministic by construction — zero cross-seed instability. It scales to both corpora, and correcting the modality gap is the decisive fix, not the solver.

  • Fixed dictionary — no estimation, no seeds, no instability.
  • Same output schema as the SAE — the same judge can score it.
  • Honest caveat — frequent terms are "mixed", a property of the supplied dictionary.

Impl: src/concept_discovery/spliece.py — decompose_image()

SPLiCE trades estimation for a given dictionary. No seed, no non-identifiability. SAE-compatible schema → same judge scores it (slide 16).

Caveat: frequent terms "mixed" (ROCOv2 imaging modalities; IU bilingual/off-topic) — a dictionary property, not SPLiCE's.

Part 3 of 3 · Carmine Francesco Benvenuto

Concept organisation · Gap 5

Dense, but weakly separated.

We cluster the active vocabulary into concept families with RadLex ancestors and two degeneracy guards — leaf-root rejection, and a subtree canopy that rejects overly generic ancestors (e.g. "anatomical entity"). SPLiCE, the densest, cuts concepts-per-image ~1.75× — but silhouette is weak (~0.02): the noisy vocabulary makes every cluster inherit a noisy ancestor.

Impl: src/concept_discovery/organize.py — cluster_concepts(), annotate_radlex()

Organisation does its job (1.75× redundancy reduction) but separation is weak because the IU vocabulary is noisy.

Standalone stage — does not feed the judge.

Faithfulness · Gap 2

A small upper tail is genuinely faithful.

Do any features track real pathology? Correlating activations against the ground-truth labels (with a shuffle null), a small upper tail is genuinely faithful — the strongest looks like a mass, clinically plausible. But those same features stay unstable across seeds: partially faithful, entirely non-reproducible.

Real signal exists (~18% upper tail; "mass" feature |r|=0.459), but small and unstable. Not "concepts work" — a weak, non-reproducible tail.

Part 3 of 3 · Carmine Francesco Benvenuto

Conclusions

Three lessons that generalise.

Null + permutation-invariant

Interpretability claims about discovered concepts must be calibrated against analytical nulls and a permutation-invariant statistic. Slot-wise overlap is degenerate.

Scale is the last suspect

For a failing factorisation, data scale is the least likely culprit: 10× changed training health, not identifiability.

Partially-negative evaluation

An honest, partially-negative evaluation is more useful than a curated one. The defect is localised (representation site), not global.

Ref: main.tex §6 — 4 contributions: relabeling control · matched metric + null · scale refutation · partially-negative eval

Synthesis: we did not "break" the pipeline — we showed where and why SAE concepts are not identifiable, localised in the representation site.

SPLiCE = only deterministic, judge-ready alternative; ~18% upper tail faithful; organisation weakly separated.

Limitations & future work

SAE on the patch-token residual stream.

The diagnosis is predictive: decomposing BiomedCLIP patch-token residual stream (a mid layer, not the pooled CLS) should restore identifiability — the experiment our characterisation points to as the next step.
