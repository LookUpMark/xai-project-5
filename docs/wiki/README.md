# Wiki

This folder serves as the reference knowledge base and API documentation for the project's codebase.

**What to include here:**
- Documentation for configuration dataclasses (`CONFIG.md`).
- Overviews of packages and their APIs.
- Subdirectories for specific modules (e.g., `autoencoder/`).

**Usage:**
Update these documents when the codebase's public API, configurations, or core contracts change. This is for technical reference, not for strategy or planning.
